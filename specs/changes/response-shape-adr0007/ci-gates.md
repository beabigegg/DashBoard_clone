# CI/CD Gate Review — response-shape-adr0007

## Required Gates for This Change

| gate | tier | required | trigger | command/workflow | artifact |
|---|---:|:---:|---|---|---|
| contract-validate | 0 | yes | local pre-PR | `cdd-kit validate` | — |
| response-shape-validate | 1 | yes | push / PR | `cdd-kit validate --contracts` (new step in `contract-driven-gates.yml`) | — |
| openapi-sync | 1 | yes | push / PR | `cdd-kit openapi export --check --out contracts/openapi.json` (existing step) | `contracts/openapi.json` |
| unit-mock-integration | 1 | yes | push / PR | see ci-gate-contract.md `unit-mock-integration` row | junit XML |

`cdd-kit validate --contracts` validates that every entry in
`tests/contract/response-samples.json` matches the typed schema declared under
`## Schemas` in `contracts/api/api-contract.md`. Concretely:

- Loads `contracts/openapi.json` (generated by the preceding openapi-sync step).
- Iterates all 158 sample entries; for each, loads `tests/contract/samples/<file>.json`,
  drills to `dataPath` when present (envelope unwrap), and validates the object
  against the resolved JSON Schema.
- Fails (exit 1) on any field-name or type mismatch. Endpoints whose contract
  `response schema` cell is still prose are skipped (advisory-only adoption path).

Local pre-PR command (Tier 0, runs before opening a PR):

```bash
cdd-kit validate --contracts
```

This gate is **required** (merge-blocking, AC-8 from change-classification.md).

## CI/CD Workflow

File: `.github/workflows/contract-driven-gates.yml`, job `contract-and-fast-tests`.

Add the following step immediately after the existing `OpenAPI sync gate` step:

```yaml
- name: Response-shape contract gate
  run: cdd-kit validate --contracts
```

No new job is needed; both the openapi-sync and response-shape steps belong in
the same `contract-and-fast-tests` job so they share the same cdd-kit install and
checked-out `contracts/` tree. The `openapi-sync` step must precede this step
because `cdd-kit validate --contracts` reads the freshly exported `contracts/openapi.json`.

No other workflow files require changes for this gate.

## Promotion Policy

`response-shape-validate` starts as a **required** Tier 1 gate per AC-8. No
informational incubation period is needed: the gate is deterministic
(schema-vs-sample comparison, no real infrastructure) and has zero flakiness risk.

Per `ci-gate-contract.md` Required Check Policy: once wired as required, gate
failures block merge immediately with no grace period.

## Rollback Policy

- If `cdd-kit validate --contracts` fails after merge (e.g. a follow-on PR
  introduces a schema regression): revert the schema change in
  `contracts/api/api-contract.md` and regenerate `contracts/openapi.json` in a
  fix PR. No parquet cleanup or service restart required — this gate is
  contract-metadata only; no production source is modified (AC-7).
- If a sample in `tests/contract/samples/` becomes stale after a route change:
  re-run `tests/contract/capture_samples.py` against the Flask test client and
  commit the updated sample. No data loss or downtime risk.
