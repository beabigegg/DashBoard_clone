- Standard policy applies (`contracts/ci/ci-gate-contract.md` § Informational Gate Promotion Policy, referenced not duplicated): 20 calendar days or 60 runs, pass rate above threshold, failures triaged, runtime acceptable, owner assigned.
- `playwright-critical-journeys`/`playwright-resilience`/`playwright-data-boundary` keep their pre-existing Tier-1-required status (only newly *wired*, not newly *promoted*).

## Rollback Policy
Formalizes design.md § Migration/Rollback into an ordered runbook:
1. **MySQL DDL**: the 3 new tables (`scripts/sql/production_achievement_tables.sql`) are manually applied before deploy, same precondition as the existing 2 PA tables. On rollback they are left orphaned — dropping them is optional cleanup, never required.
2. **Parquet fast-forward (optional)**: `_PA_SPOOL_SCHEMA_VERSION` 1→2 self-heals via key mismatch (stale v1 parquets age out by TTL, never re-hit). `rm -f tmp/query_spool/production_achievement/*.parquet` is available if operators want them gone immediately instead of waiting.
3. **Chart cutover (hard, no flag)**: no `*_USE_UNIFIED_JOB`-style toggle exists for the frontend chart — rollback is a full PR revert, restoring `AchievementChart.vue` from git history and removing `PlanAchievementStackedChart.vue`.
4. **Settings-page registry (7 locations, order matters)**: revert `vite.config.ts`'s `INPUT_MAP` entry first or atomically with the rest — a dangling entry pointing at deleted source blocks the frontend build/boot. The other 6 (`routeContracts.js`, `navigationState.js` `STANDALONE_DRILLDOWN_ROUTES`, `nativeModuleRegistry.js`, `page_status.json`, `route_scope_matrix.json`, `asset_readiness_manifest.json`) degrade gracefully but should revert in the same commit to avoid nav-parity test failures.
5. **Warm-cache**: remove the 2 new `_WARMUP_JOBS` entries in `spool_warmup_scheduler.py`, else the scheduler keeps warming a removed cache path. No hard-failure risk regardless (D5: miss always degrades to the existing 202-poll path).
6. **Contracts**: after reverting `api-contract.md`'s schema-version bump, re-run `cdd-kit openapi export --check` for both `contracts/openapi.json` and `contracts/api/openapi.json` — `openapi-sync-gate` re-fires automatically on the revert push.
7. **No RQ worker unit** to stop/disable or `deploy/*.service` to touch — confirmed no new worker service by change-classification.md.
