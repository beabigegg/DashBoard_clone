## Workflow Changes Applied

No CI workflow files are added or modified. All gates execute via existing `npm run test:e2e` script
and the shared Chromium browser at `~/.cache/ms-playwright/` (do NOT run `playwright install`).
`PLAYWRIGHT_CHROMIUM_EXECUTABLE_PATH` must be set in the runner environment to point at the shared
binary; `E2E_BASE_URL` must resolve to a live app instance (GunicornHarness or equivalent).

Concurrency: existing PR workflow `concurrency` group applies; no change needed.

## Promotion Policy

A PR is promotion-eligible when:

1. `static-scope-check` passes — confirms zero edits outside `frontend/tests/playwright/`.
2. `e2e-targeted` passes with 0 failures across all 9 repaired spec files.
3. `e2e-full-suite` reports exactly 130 passed / 0 failed / 3 skipped.
4. qa-reviewer attestation recorded in `agent-log/qa-reviewer.yml` confirming no assertion was
   weakened or skipped to reach green (test-plan.md § Test-Discipline Constraints).

No tier-3 nightly or tier-4/5 gates required; this change is test-only, low-risk, isolated.

## Rollback Policy

All edits are confined to `frontend/tests/playwright/` spec files. If any gate fails post-merge:

- Revert via `git revert <merge-sha>` — no production artifact, parquet schema, or spool key is
  affected; rollback is instantaneous with zero data impact.
- No cache flush, parquet schema bump, or service restart required.
- Re-open change as a new `fix-e2e-spec-failures-v2` change-id if root cause is new.

## Merge Eligibility

mergeable when all three Required Gates are green and qa-reviewer attestation is present.
