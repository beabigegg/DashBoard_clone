# PRD — MES Dashboard 報表系統產品需求文件

> 版本：1.0 | 日期：2026-04-08 | 狀態：正式版

---

## 1. 背景與目標

### 1.1 專案背景

製造執行系統（MES）累積了大量的生產、設備、品質與物料數據，存放於 Oracle 資料庫。  
工廠各部門（生產、品質、設備工程）需要定期查詢這些數據進行決策，但：

- 直接查資料庫需要 SQL 能力，門檻高
- 既有報表工具更新慢、無法即時查詢
- 重查詢耗時長，影響其他使用者
- 多人同時查詢時系統容易過載

MES Dashboard 旨在提供一個穩定、快速、易用的 Web 報表平台，讓各部門工程師能夠自助查詢生產數據。

### 1.2 專案目標

| 目標 | 說明 | 量化指標 |
|------|------|----------|
| 查詢自助化 | 工程師無需 SQL 即可查詢 MES 數據 | 16 個功能性報表頁面 |
| 系統穩定性 | 在多人並發使用下保持服務可用 | 支援 50 人使用 / 10-12 人同時在線 |
| 查詢效能 | 常用數據快速回應，重查詢非同步處理 | 快取命中 ≤ 2s；重查詢非同步不卡介面 |
| AI 輔助查詢 | 自然語言轉 SQL，降低查詢門檻 | Text2SQL / Function / Agent 三模式 |
| 異常偵測 | 自動偵測生產異常，主動預警 | 每日排程執行，結果推送至儀表板 |

### 1.3 專案範圍

**涵蓋：**
- 16 個 MES 數據報表頁面（WIP、Hold、設備、品質、良率、材料追溯、生產歷史等）
- AI 智能查詢助手（自然語言 → SQL）
- 異常偵測排程系統
- 系統管理儀表板（快取、效能、用量 KPI）
- 使用者認證（LDAP）與角色管理（admin / user）

**不涵蓋：**
- 直接修改 MES 資料庫數據
- MES 系統本身的製程控制功能
- 報表排程訂閱與 Email 推送
- 行動裝置（Mobile）專屬 UI 優化

## 2. 使用者與角色

### 2.1 使用者角色

| 角色 | 職責 | 典型使用場景 |
|------|------|-------------|
| 一般使用者 (User) | 生產線工程師、品質工程師、設備工程師 | 查詢 WIP 狀態、分析不良原因、追溯批號歷史 |
| 管理員 (Admin) | 系統管理員、IT | 管理快取、監控效能、調整功能開關、查看系統日誌 |

### 2.2 使用者規模預估

| 環境 | 總使用者 | 同時在線 | 硬體需求 |
|------|----------|----------|----------|
| DEV | 1 人 | 1 人 | 2+ cores, 8GB RAM |
| PRD | 50 人 | 10-12 人 | 4+ cores, 12-16GB RAM, SSD 50GB+ |

| 功能編號 | 功能名稱 | 說明 |

### 3.11 M10 — 認證與權限

| 功能編號 | 功能名稱 | 說明 |
|----------|----------|------|
| M10-F01 | LDAP 認證 | 透過 LDAP API 驗證使用者帳號密碼 |
| M10-F02 | Session 管理 | Session 存活時間可配置（預設 8 小時） |
| M10-F03 | Admin 角色 | 特定 Email 帳號自動取得 Admin 權限 |
| M10-F04 | 本地認證（開發用） | 可配置帳號密碼，僅 DEV 環境開啟 |
| M10-F05 | 登入紀錄 | 記錄登入/登出事件至 SQLite |

### 3.12 非功能性需求

| 項目 | 要求 |
|------|------|
| 快取命中回應時間 | ≤ 2 秒 |
| 重查詢（非同步） | 任務提交 ≤ 1 秒，輪詢間隔 3-5 秒 |
| 並發支援 | 10-12 人同時在線，正式環境 4 Workers |
| 系統可用性 | Watchdog 自動重啟，Circuit Breaker 保護資料庫 |
| 記憶體安全 | Worker RSS 超限自動回收，系統記憶體超 90% 拒絕重查詢 |
| 安全性 | CSRF、CSP、HSTS、XSS 防護、SQL 參數化 |
| 相容性 | 現代瀏覽器（Chrome / Edge）；不支援 IE |

### 整體驗收標準

- [ ] 所有 P0 功能模組正常運作（M1、M2、M3、M4、M10）
- [ ] 系統在 10 人同時在線下保持回應正常
- [ ] `/health` 端點回傳 200 且顯示各元件健康狀態
- [ ] 所有安全防護機制正常啟用（CSRF Token 驗證、CSP 標頭）
- [ ] 非同步任務（reject-history、trace、production-history 等）可正確提交與取回結果
- [ ] 管理員可透過儀表板監控系統狀態並執行基本維護操作
